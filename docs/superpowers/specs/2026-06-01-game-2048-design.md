## 概述

在 XXGame 网站新增 `/game/2048` 路由，实现经典 2048 数字滑块游戏。纯前端运行，无后端依赖。最高分通过 localStorage 持久化。

## 文件结构

```
src/
├── app/game/2048/
│   └── page.tsx          # "use client" 页面组件 — 棋盘渲染、键盘事件、计分UI
└── lib/
    └── game-2048.ts      # 纯函数游戏逻辑 — 无副作用、无状态
```

## 数据流

```
键盘事件 (Arrow keys)
  → move(board, direction) 返回 { newBoard, scoreGain }
  → 若棋盘变化: addRandomTile(newBoard) + 更新 score
  → 更新 localStorage bestScore
  → setState 触发重渲染
```

## game-2048.ts API

| 函数 | 签名 | 说明 |
|------|------|------|
| `createBoard()` | `() => Board` | 4×4 全零棋盘，预放 2 个 tile (2 或 4) |
| `move()` | `(board: Board, dir: Direction) => MoveResult` | 移动+合并，返回新棋盘和本轮得分 |
| `addRandomTile()` | `(board: Board) => Board` | 随机空位放 2 (90%) 或 4 (10%) |
| `canMove()` | `(board: Board) => boolean` | 任何方向有合法移动 → true |
| `hasWon()` | `(board: Board) => boolean` | 存在 2048 或更大的 tile |

类型:
```ts
type Board = number[][]       // 4×4, 0 表示空格
type Direction = "up" | "down" | "left" | "right"
type MoveResult = { board: Board; score: number; moved: boolean }
```

## 边界情况处理

- **无效移动**: 若 `moved === false`，不更新棋盘、不生成新 tile、不减分
- **连胜后继续**: 达成 2048 后弹出提示，用户可选继续游戏
- **无空位 + 无可合并** → Game Over
- **localStorage 不可用**: try-catch 静默降级，功能不受影响
